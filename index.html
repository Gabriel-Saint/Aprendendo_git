1 git init 2 git status para ver quais arquivos não estao trackeados 3 git add . adiciona geral 4 git status para verificar se foi trackeado 5 git commit -a(add todo tipo de arquivo para o commit) -m(add uma mensagem para nortear e indentificar o que foi feito) para 6 executando 1° teste no gitlab errado 1° teste no github git switch -c nome do branch cria um novo e ja alterna para ele git push enviar git pull trazer as alterações do repositório remoto para o seu repositório local. git checkout branch2 git merge branch1 integra as alterações do 1 no 2 git log historico de commits git remote -v mostra os repositórios remotos configurados no seu projeto. Isso inclui o nome dos repositórios remotos e seus URLs. git branch mostra a lista de branches (ramificações) disponíveis no repositório e destaca o branch atual com um asterisco.
